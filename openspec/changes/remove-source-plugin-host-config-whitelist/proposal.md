## 背景

源码插件属于随宿主编译和交付的受信任扩展，读取宿主配置应当服务于框架级扩展能力，而不是受管理工作台或动态插件授权模型中的公开 key 白名单限制。当前 `HostServices.HostConfig()` 只允许读取少量显式公开 key，导致源码插件无法按需读取完整宿主配置。

## 变更

- 移除源码插件宿主配置读取适配器中的公开 key 白名单。
- 明确源码插件可通过 `HostServices.HostConfig()` 读取任意非根宿主配置 key。
- 保持动态插件 `hostServices` 授权快照校验不变，动态插件仍需按 manifest 授权访问宿主配置 key。

## 影响

- 源码插件宿主配置读取能力变为不受 key 白名单限制的只读能力。
- 不新增 HTTP API、数据库表、运行时配置写入能力或前端用户可见交互。
