## ADDED Requirements

### Requirement: 插件生命周期 E2E 必须显式满足被测插件依赖

插件生命周期 E2E 测试在安装、启用或确认授权一个存在插件依赖的被测插件前，SHALL 通过幂等前置条件确保必需依赖插件已经安装。测试不得依赖其他测试文件留下的插件安装状态，也不得把缺失依赖造成的授权确认按钮禁用表现为长时间 UI 超时。

#### Scenario: 安装存在插件依赖的被测插件
- **WHEN** E2E 测试需要安装或启用一个声明了必需插件依赖的被测插件
- **THEN** 测试必须先查询依赖插件安装状态
- **AND** 依赖插件未安装时必须通过幂等辅助安装依赖插件并确认其安装完成
- **AND** 依赖安装完成后才允许打开被测插件授权弹窗或调用被测插件安装接口

#### Scenario: 依赖安装失败快速暴露业务错误
- **WHEN** 依赖插件安装或被测插件安装接口返回业务失败
- **THEN** 测试必须输出接口错误响应作为失败原因
- **AND** 不得继续等待因依赖缺失而被禁用的 UI 确认按钮直到超时

#### Scenario: 单文件运行保持自包含
- **WHEN** 开发者单独运行插件生命周期 E2E 文件
- **THEN** 测试必须自行准备被测插件所需的依赖插件状态
- **AND** 不得要求完整 E2E 套件中的其他文件预先安装该依赖插件
