## Why

LinaPro 后端在 GoFrame v2 约定、API 合同、源码组织、运行期依赖管理和启动治理上积累了多类系统性偏差。部分 API 直接暴露数据库实体，导致密码、软删除时间、存储路径、哈希和内部租户治理字段存在被动外泄风险；部分公开时间字段以格式化字符串、`time.Time` 或 `gtime.Time` 直接穿透响应边界，无法为前端页面和用户语言环境保留统一的本地格式化能力；跨模块稳定枚举重复定义，也让 DTO、控制器与服务层之间的契约更容易漂移。

与此同时，宿主 Controller、Middleware、Service、插件宿主适配器和 WASM host service 在业务构造函数、请求路径或插件回调中再次调用 `service.New()`，让权限、会话、插件启用态、运行时配置、i18n 与缓存协调等高风险路径存在实例分裂隐患。再叠加事务内错误吞掉、启动期重复扫描和无差异同步、主文件职责混杂、接口注释不足、文件顶部说明过于空泛等问题，代码质量、运行时稳定性和审查可控性都受到了持续影响。

项目当前没有历史兼容包袱，数据库初始化、内部函数签名、启动装配和插件注册路径都可以直接收敛到更严格的治理基线。本次变更的目标，是把 API 边界、源码组织、依赖注入、缓存一致性、宿主运行能力和启动性能统一纳入同一套代码质量规范，并同步完成必要的实现整改。

## What Changes

### API 合同与响应边界治理

- 统一后端 API 的 REST 语义、资源路径风格和路径参数绑定规则，约束读取使用 `GET`、创建使用 `POST`、更新使用 `PUT`、删除使用 `DELETE`，并统一输入 DTO 使用 `json` 标签。
- 强化 API 文档标签规则，要求所有可见输入输出字段提供清晰的 `dc` 与 `eg`，并在枚举、可选筛选项和跨模块稳定值类型场景中给出完整契约说明。
- 为用户、角色提供 RESTful 批量删除接口，在单事务内复用单条删除保护规则，并同步调整前端批量操作调用方式。
- 规定公开 HTTP JSON 响应中的瞬时时间字段统一返回 Unix 毫秒时间戳，明确 `date-only` 日历日期字段例外，并通过共享投影 helper 迁移宿主与源码插件现有响应字段。
- 将宿主与源码插件 API 响应中的 `entity.*` 暴露改为显式响应 DTO，禁止响应边界直接嵌入数据库实体或把实体指针直接赋值到响应字段。
- 抽象 `listorder`、`tenantoverride`、`statusflag` 等跨模块稳定公共枚举契约，并直接复用已有稳定公共组件，避免引入大一统 `pkg/enums` 包。

### 后端实现与源码治理

- 收敛 GoFrame v2 ORM、软删除和事务约定，禁止手写软删除过滤与时间字段维护，统一使用 DO 对象传递写入数据。
- 明确生产后端 `panic` 治理边界，仅保留启动、顶层注册、`Must*` 和未知 panic 重抛等允许场景，其余运行期错误改为显式返回 `error`。
- 将宿主、源码插件和 `lina-core/pkg` 公共组件的主文件统一治理为契约入口：主文件只保留组件说明、核心类型、接口定义、默认实现结构体、编译期断言和构造函数，复杂实现迁移到职责文件。
- 强化接口方法注释与文件顶部注释质量，要求注释覆盖关键输入、输出、错误和权限、数据权限、缓存、i18n、事务、并发等适用约束。
- 为 `linactl` 命令实现补充 `command_<command>.go` 命名约束和 `internal/<component>/` 子组件组织规则，避免根目录继续堆积复杂共享实现。

### 显式依赖注入与共享实例治理

- 将宿主与源码插件的 Controller、Middleware、Service、插件宿主服务适配器和 WASM host service 的运行期依赖改为构造函数逐项显式传入，禁止在业务构造函数、请求路径、插件回调或 host service 调用路径中隐式创建关键服务图。
- 禁止使用 `Dependencies`、`Deps`、`Options` 等聚合结构体整体传递多个接口型运行期依赖，要求依赖变化通过编译错误暴露所有未同步调用点。
- 要求初始化与注册 API 在依赖缺失、参数非法、配置缺失或后端创建失败时返回 `error`，由调用栈最上层决定是否中止进程。
- 对认证、会话、权限、插件治理、运行时配置、i18n、cachecoord、kvcache、锁、通知、插件运行时缓存和插件宿主适配器等缓存敏感组件，强制共享启动期实例或共享后端，避免单节点孤立状态。
- 通过源码插件 registrar 向插件发布宿主能力目录，由宿主统一构造 `pkg/pluginservice/*` 适配器，并在 HTTP、全局中间件、Cron 与 WASM host service 路径中复用。

### 数据一致性、启动性能与宿主运行能力

- 修复用户删除、角色删除、角色授权用户和菜单删除中的事务一致性问题，保证关联清理失败会回滚整笔操作，且访问拓扑通知仅在事务提交后触发。
- 为用户、用户角色、角色菜单、在线会话和定时任务相关表补齐索引，移除 `sys_job` 外键约束并改由应用层维护一致性，对字典表补齐软删除字段。
- 将菜单子树判定改为一次加载 + 内存遍历，降低层级查询的 SQL 往返成本。
- 新增匿名健康检查接口、复用 GoFrame `Server.Run()` 的优雅停机流程、将上传访问路由收口到文件模块受保护路径，并将内置定时任务默认时区改为配置项。
- 引入一次启动编排内共享的 startup snapshot 与差异驱动同步逻辑，消除无差异插件同步的空事务、重复全表读取和默认 SQL 明细日志噪声。

### 前端运行协同与模块边界

- 为服务器监控页和用户消息未读轮询增加页面可见性感知，隐藏页签时暂停轮询、恢复可见时立即刷新一次。
- 将路由守卫中的 `loadedPaths` 收敛为有界 LRU，避免长时间会话中无界增长。
- 语言切换仅同步与语言强相关的本地状态，不再重新拉取整套权限、菜单和路由数据，菜单标题通过本地 i18n key 响应式更新。
- 明确模块禁用的后端降级边界，只影响功能暴露，不破坏历史数据完整性。

## Capabilities

### New Capabilities

- `host-runtime-operations`：新增匿名健康检查、优雅停机、上传访问统一鉴权和可配置调度默认时区。
- `startup-sql-efficiency`：新增启动快照复用、差异驱动同步、无差异零写入路径和结构化启动摘要日志。
- `service-dependency-injection-governance`：新增显式依赖注入、共享实例、初始化错误返回和静态治理校验规则。

### Modified Capabilities

- `api-contract-consistency`：补充 REST 语义、路径参数绑定、文档标签、批量删除、时间戳响应契约、响应 DTO 加固与公共枚举契约治理。
- `backend-conformance`：补充 GoFrame ORM/软删除规范、事务与 panic 治理、主文件职责、接口方法注释和文件顶部注释治理。
- `distributed-cache-coordination`：要求缓存敏感服务共享实例或共享后端，并将实例来源纳入缓存一致性审查。
- `plugin-host-service-extension`：要求插件宿主服务适配器由宿主统一构造并复用。
- `plugin-http-slot-extension`：要求源码插件 HTTP、中间件和 Cron 注册回调通过宿主发布依赖构造控制器和服务。
- `plugin-manifest-lifecycle`：要求插件清单同步在无差异时不产生数据库副作用。
- `plugin-startup-bootstrap`：要求同一启动编排复用插件治理快照，避免重复全表扫描。
- `user-management`、`role-management`、`menu-management`、`online-user`：补充事务一致性、索引和批量操作能力。
- `server-monitor`、`user-message`：补充页面可见性感知轮询策略。
- `spec-governance`、`module-decoupling`：补充主规范结构校验、归档残留治理和模块禁用降级约束。

## Impact

- **后端代码**：影响宿主 `internal/controller`、`internal/service`、`internal/cmd`、`pkg` 公共组件，以及源码插件 `backend/api`、`backend/internal/service`、`backend/plugin.go` 等运行期依赖装配与响应边界。
- **SQL 与配置**：影响用户、角色、菜单、在线会话、定时任务与字典相关 SQL，新增 `health.timeout`、`shutdown.timeout`、`scheduler.defaultTimezone`，并将默认 SQL debug 配置设为关闭。
- **前端**：影响用户、角色、服务器监控、消息轮询、语言切换、时间字段展示和批量删除调用。
- **测试与审查**：新增或更新事务回滚、批量删除、启动效率、显式依赖注入、API DTO 边界、健康检查与轮询行为等测试，并将相关规则纳入 `lina-review`。
- **i18n 影响**：运行时 UI 语言包和 manifest 运行时 i18n 资源不因本次治理新增语言项；宿主与源码插件的 apidoc i18n 资源会随着时间字段迁移和响应字段收口同步调整。
- **兼容性影响**：对外 API 的资源路径和大多数语义保持稳定，但公开瞬时时间字段统一变为数字毫秒时间戳，响应不再暴露内部实体字段；对内构造函数签名与插件注册依赖传递方式属于治理性 breaking change。
